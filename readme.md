# **img2svg: A Fully Localized, Modular Pipeline for Automatic Conversion of Raster Images to Editable Vector Graphics**

## **研究方法**

本研究提出了一种名为 img2svg 的完全本地化图像矢量化方法，旨在将任意栅格图像（如 JPG、PNG）自动转换为结构清晰、可编辑的 SVG 矢量图形。该方法不依赖任何云端大模型 API，完全基于开源计算机视觉与图像处理库实现，具有低延迟、零成本、数据隐私安全等优势。整体流程由八个核心模块组成，各模块协同工作，实现对图像内容的理解与结构化重建。

**图像加载与预处理模块**
首先利用 Pillow 库加载图像，自动校正 EXIF 旋转信息。若图像包含透明通道（RGBA/LA/P 模式），则将其与白色背景合并，确保后续分析基于不透明 RGB 图像。为避免过大图像影响处理效率，将图像最长边缩放至 2048 像素以内，同时保持原始宽高比。预处理后的图像同时以 PIL Image 对象和 NumPy 数组形式传递给后续模块。

**颜色分析模块**
采用 K-Means 聚类算法提取图像主色调。将图像像素重塑为三维向量，使用 scikit-learn 实现的高效 K-Means（若无 scikit-learn，则回退至自定义 NumPy 实现）对像素进行聚类，聚类数默认为 12。聚类中心经裁剪后转换为十六进制颜色字符串，并按像素覆盖面积排序，构成图像的调色板。此外，通过局部像素采样（排除近白色背景）精确获取任意坐标的颜色，为面板、文字及图标提供准确的着色依据。

**版面检测模块**
基于 OpenCV 实现面板区域检测。首先将图像转换为灰度图，应用自适应阈值生成二值图，再通过形态学闭运算连接邻近区域。提取所有外部轮廓，依据面积、长宽比和位置过滤掉过小或过大的区域，并通过交并比去重，保留候选面板。对每个候选面板，分别采样其顶部（可能为标题栏）和主体区域的像素颜色，通过亮度差异判断是否具有彩色标题栏，从而确定面板的 `has_header` 属性。所有面板信息以结构化列表形式保存。

**环形元素检测模块**
针对常见的环形图表（如 DMTA 循环），设计专用检测算法。首先对灰度图进行高斯模糊，应用霍夫圆变换寻找候选圆。选取最接近图像中心且半径较大的圆作为中心环。通过从圆心沿半径向外扫描像素亮度变化，估算环的内外半径。为识别环上的分段，沿圆周等角度采样颜色，将连续颜色相近的角度区间聚类为独立段，最多保留四个主要段，并映射至预设标签（“Design”、“Make”、“Test”、“Analyze”）。同时检测中心区域是否存在复杂图案（如机器人图标），为后续渲染提供依据。

**OCR 文字识别模块**
集成 Tesseract OCR 引擎识别图像中的文字内容。对预处理后的图像进行 OCR 识别，提取每个文字区域的文本、坐标、宽度、高度及置信度。通过分析区域内像素亮度，选取最深色作为文字颜色，并根据字符宽度与高度比例估算字体粗细。识别结果将根据位置分配给对应面板（标题栏或正文）或作为独立文字保留。

**图标检测与类型推断模块**
在每个面板内部，根据面板标题文本和边缘特征推断可能存在的图标类型。将面板内除标题栏和底部文字外的区域提取出来，计算边缘密度和圆形检测结果，并结合标题关键词（如“knowledge”对应“book”、“lab”对应“flask”）匹配预定义的图标类型库。每个图标区域生成包含类型、位置、尺寸和主副颜色的描述字典，供后续渲染使用。

**SVG 渲染模块**
将所有分析结果整合为符合 SVG 标准的文档。渲染遵循以下原则：（1）所有文字均使用 Times New Roman 字体，以 `<text>` 节点独立呈现，确保可编辑性；（2）每个图标封装为独立的 `<g>` 组，并赋予唯一 `id`，可在矢量编辑软件（如 Adobe Illustrator、Inkscape）中整体移动、缩放或修改颜色；（3）面板背景使用线性渐变模拟光照效果，标题栏采用其采样颜色生成的渐变；（4）环形分段以路径绘制，并添加阴影、高光及箭头指示；（5）整体图层结构清晰，便于后期人工调整。

**EPS 导出模块**
为满足出版和印刷需求，提供 EPS 格式输出。通过解析 SVG 路径，将其转换为 PostScript 指令，并嵌入颜色定义与字体引用，生成与 Adobe Illustrator 完全兼容的 EPS 文件。

**验证与评估**
每个生成的文件均通过 XML 解析验证结构完整性。在开发过程中，对多类图像（包括科研插图、流程图、海报等）进行了测试，统计面板检测准确率、文字识别召回率及图标匹配准确率，结果表明该方法在常见学术图像上具有良好的鲁棒性与重建质量。

## 研究限制：

| 能力               | 开源 VLM | 说明                                             |
| ------------------ | -------- | ------------------------------------------------ |
| 理解图像内容       | 支持     | Qwen-VL / LLaVA 都很强                           |
| 精确颜色采样       | 支持     | 纯 Pillow 像素采样，不需要 AI                    |
| 真实 `<text>` 节点 | 支持     | OCR (PaddleOCR/EasyOCR)                          |
| 可移动 icon 组     | 支持     | 规则引擎 + VLM 描述                              |
| 批量处理           | 支持     | 完全支持                                         |
| **完全复原复杂图** | 不太行   | VLM 理解深度比 Claude 弱，需要更好的 prompt 工程 |

**最佳免费方案：Qwen2-VL（阿里，最强开源 VLM）+ PaddleOCR + Pillow**

| 开源 VLM 做的事 | 本地 CV 算法替代                                             |
| --------------- | ------------------------------------------------------------ |
| 理解图像布局    | OpenCV 轮廓检测（自适应阈值 + 形态学）识别面板、区域，通过面积、长宽比过滤 |
| 识别颜色        | K-Means 像素聚类提取主色调，结合局部像素采样（排除近白背景）获取精确颜色值 |
| 识别文字        | Tesseract OCR 引擎识别文本内容、位置、尺寸，通过像素亮度确定文字颜色 |
| 检测圆环 / 弧段 | 霍夫圆变换定位中心圆，沿圆周等角度采样颜色，聚类连续色块识别分段（DMTA 四段） |
| 推断 icon 类型  | 基于边缘密度（Canny）、圆形计数（霍夫圆）及面板标题关键词匹配预定义图标库 |

**说明**：本方法完全基于开源计算机视觉库（OpenCV、scikit-learn、Tesseract）实现，不依赖任何云端大模型 API，在颜色精确性、文字识别准确率及处理速度上均优于通用 VLM，且具备隐私安全和零成本优势。

### 立即运行（零 API，零费用）

```bash
pip install Pillow numpy opencv-python-headless scikit-learn pytesseract
sudo apt install tesseract-ocr   # Linux

python img2svg.py Fig1.png
python img2svg.py folder/images/ --output-dir vectors/
```

### 如果你有好的 GPU（想用真 VLM）

# GPU >= 8GB 显存
